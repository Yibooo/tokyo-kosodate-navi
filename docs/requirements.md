## 3. フェーズ別機能詳細

### Phase 0：基盤構築 ✅ 完了

**[P0-1] 制度データパイプライン**
- スクレイピング対象：国（こども家庭庁・厚生労働省等）/ 東京都（福祉局）/ 23区（各公式サイト）
- Claude API がHTMLから制度情報を抽出し、構造化JSONとして保存
- 抽出項目：制度名 / 対象条件（年齢・所得・子の順番）/ 給付額 / 申請方法 / リンク / カテゴリ / 有効期限
- 管理者がAPIエンドポイント叩きで手動実行（Cronは将来対応）
- 実行後、管理者へメール通知（Resend）

**[P0-2] 管理者レビュー画面** `/admin/review`
- ドラフト制度の一覧表示（承認待ち / 承認済み / 却下済みのタブ）
- 管理者が「承認 / 却下」の2アクションで確定
- 承認分のみ本番DBへ反映される仕組み

**[P0-3] 制度マスタDB設計**（Supabase PostgreSQL）

```sql
-- 制度テーブル
policies (
  id UUID PRIMARY KEY,
  layer TEXT,          -- 'national' | 'tokyo' | 'ward'
  ward TEXT,           -- 区名（国・都は null）
  name TEXT,
  summary TEXT,
  category TEXT,       -- 8カテゴリ（後述）
  amount_monthly INT,  -- 月額（円）
  amount_lump INT,     -- 一時金（円）
  amount_note TEXT,    -- 補足説明
  apply_url TEXT,
  source_url TEXT,
  valid_from DATE,
  valid_to DATE,
  status TEXT          -- 'draft' | 'approved' | 'rejected'
)

-- 条件テーブル
policy_conditions (
  policy_id UUID REFERENCES policies(id),
  income_max INT,      -- 所得制限（万円）
  child_age_min INT,   -- 対象年齢 最小（月）
  child_age_max INT,   -- 対象年齢 最大（月）
  birth_order_max INT  -- 第何子まで
)

-- スクレイピングログ
scrape_logs (
  id UUID PRIMARY KEY,
  target_name TEXT,
  target_url TEXT,
  status TEXT,
  policies_found INT,
  policies_skipped INT,
  error_message TEXT,
  started_at TIMESTAMPTZ,
  finished_at TIMESTAMPTZ
)
```

### Phase 1：MVP ✅ 完了

**[P1-1] 入力フォーム**

| 入力項目 | 形式 | 備考 |
|---------|------|------|
| 居住区 | ドロップダウン（23区・地域別グループ化） | 都心部・東部・南部・西部・北部に分類 |
| 子の生年月日 / 出産予定日 | 年月セレクター | 未出産（出産予定）も対応 |
| 第何子か | 選択（第1子〜第4子・第5子以上） | |
| 世帯年収 | スライダー（50万円単位） | 〜1,500万円 |

**[P1-2] ルールベースマッチングエンジン**（`src/lib/matcher.ts`）
- ユーザー入力条件 × 承認済み制度マスタ全件 → 該当制度リスト生成
- **ポイント：LLMは使用しない（DB直接マッチ）** → 高速・低コスト・安定
- マッチロジック：年齢範囲・所得上限・出生順位・居住区の4条件で絞り込み
- ソート：月額換算の大きい順（一時金は12で割って換算）

**[P1-3] 結果表示**（`src/components/ResultList.tsx`）
- 層フィルター（国 / 東京都 / 区 / すべて）
- カテゴリフィルター（8カテゴリ・動的生成）
- 各カードに表示：制度名 / 給付額 / カテゴリバッジ / 層バッジ / 対象条件の要約
- 展開で詳細：概要・申請先リンク・申請方法

**[P1-4] 制度カテゴリ**（8種類）

| カテゴリID | 表示名 | 例 |
|-----------|-------|---|
| `給付金・手当` | 💰 給付金・手当 | 児童手当、乳児養育手当 |
| `一時金` | 🎁 一時金 | 出産育児一時金、出産お祝い金 |
| `医療費助成` | 🏥 医療費助成 | 子ども医療費助成（都・各区） |
| `保育・教育` | 🏫 保育・教育 | 幼児教育無償化、高校授業料無償化 |
| `育児サービス` | 👶 育児サービス | ベビーシッター補助、産後ケア |
| `雇用・休業` | 💼 雇用・休業 | 育児休業給付金、産後パパ育休 |
| `税制優遇` | 📊 税制優遇 | 住宅ローン控除子育て特例 |
| `物品・生活支援` | 🚲 物品・生活支援 | 電動アシスト自転車購入費補助 |

## 5. アーキテクチャ

```
[管理者操作]                    [ユーザー操作]
     │                                │
     ▼                                ▼
POST /api/scrape              GET / (入力フォーム)
     │                                │
     ▼                                ▼
Claude API                    ユーザー条件入力
（HTML解析・構造化）                    │
     │                                ▼
     ▼                        GET /results
 scrape_logs                  （ルールベースマッチング）
     │                                │
     ▼                                ▼
policies テーブル             policies テーブル
（status='draft'）            （status='approved'）
     │
     ▼
管理者レビュー (/admin/review)
・承認 → status='approved'
・却下 → status='rejected'
```

**ポイント：ユーザーリクエストごとにLLMを呼ばない**
→ DB検索のみ → 高速（< 1秒）・コスト０・スケーラブル
