# 要件定義書 v0.1

## システム名：東京23区・子育て支援ナビゲーター

---

## 1. システム概要

| 項目 | 内容 |
|------|------|
| 目的 | 国・都・区の子育て支援制度を一元検索・表示し、申請漏れをなくす |
| 対象 | 東京23区在住の妊婦・子育て世帯 |
| 形態 | Webブラウザ（レスポンシブ対応） |
| 運営 | 個人PJ・無料提供（将来的に広告/有料モデル検討） |
| MVP対象区 | 港区・品川区・目黒区・江東区・江戸川区（5区） |

---

## 2. フェーズ定義

```
Phase 0：基盤構築（スクレイピング・DB・管理画面）
Phase 1：MVP公開（検索・表示・マッチング）
Phase 2：エンゲージメント（ログイン・TODO・通知）
Phase 3：スケール（残18区追加・収益モデル検討）
```

---

## 3. フェーズ別機能一覧

### Phase 0：基盤構築

**[P0-1] 制度データパイプライン**
- スクレイピング対象：国（こども家庭庁・厚生労働省）/ 東京都（福祉局）/ 5区（各公式サイト）
- AIがHTMLおよびPDFから制度情報を抽出し、構造化データとして保存
- 抽出項目：制度名 / 対象条件（年齢・所得・子の順番）/ 給付額 / 申請方法 / リンク / 有効期限
- 毎月1日に自動実行 → 管理者へメール通知「〇件更新候補があります」

**[P0-2] 管理者レビュー画面**
- 「AIが抽出したデータ（ドラフト）」を一覧表示
- 管理者が「承認 / 修正 / 却下」の3アクションで確定
- 承認分のみ本番DBへ反映

**[P0-3] 制度マスタDB設計**

制度テーブル（policies）:
- id / layer（国・都・区）/ ward / name / summary
- amount_monthly / amount_lump / conditions_json
- apply_url / valid_from / valid_to / status（draft/approved）

条件テーブル（policy_conditions）:
- income_min / income_max
- child_age_min / child_age_max
- child_birth_order_max / etc.

### Phase 1：MVP

**[P1-1] 入力フォーム**

| 入力項目 | 形式 | 備考 |
|---------|------|------|
| 居住区 | ドロップダウン（5区） | Phase3で23区へ拡大 |
| 子の生年月日 / 出産予定日 | カレンダーピッカー | 未出産も対応 |
| 第何子か | 選択（第1子〜第4子・第4子以上） | 5人以上は「第4子以上」に丸める |
| 世帯年収 | スライダー（100万円単位） | 上限1,500万円（制度の最高所得制限値から逆算） |

**[P1-2] LLMマッチングエンジン**
- ユーザー入力 → LLM（Claude API）→ 該当制度リスト生成
- LLMへの入力：ユーザー条件 ＋ 承認済み制度マスタ全件
- 出力形式：制度ID・判定理由・給付額の構造化JSON
- マッチング精度ログを記録（将来の改善用）

**[P1-3] 結果表示**
- 3層タブ（国 / 東京都 / 区）または統合一覧（デフォルト：統合）
- ソート：月額換算の大きい順（一時金は12で割って換算）
- 各カードに表示：制度名 / 給付額 / 対象条件の要約 / 申請先リンク
- 展開で詳細表示：概要・申請方法・注意事項
- 総受給額サマリー（年間・18歳まで累計の概算）をページ上部に表示

**[P1-4] 免責表示**
- 検索結果の上部に固定表示
- 文言例：「情報は定期的に更新していますが、最新情報は各自治体に確認ください。本サービスは申請を保証するものではありません。」

---

### Phase 2：エンゲージメント

**[P2-1] ユーザー認証**
- ログイン方式：Google / LINE / メール＋パスワード（NextAuth.js）
- 非ログインでも検索は使用可（Phase1の機能は維持）

**[P2-2] TODOリスト**
- 検索結果からワンタップで「申請TODO」に登録
- TODO項目：制度名 / 申請期限 / ステータス（未着手・申請中・受給済）

**[P2-3] プッシュ通知 / リマインダー**
- 申請開始日・締切日の事前通知（3日前・当日）
- LINE通知 or メール通知（ログイン方式に連動）

**[P2-4] 受給シミュレーター**
- 累計受給額グラフ（0歳〜18歳のタイムライン）
- 「受給済」チェックで実績管理
- 月ごとの「家計補助額」表示

---

### Phase 3：スケール

- 残り18区のスクレイピング対象追加
- アクセス状況次第で広告・プレミアムモデル検討
- 多言語対応（英語）検討

---

## 4. 非機能要件

| 項目 | 要件 |
|------|------|
| レスポンシブ | スマホ・PCどちらでも使用可 |
| 表示速度 | 検索結果表示 3秒以内（LLMレスポンス含む） |
| データ鮮度 | 毎月1日更新・承認後即反映 |
| 可用性 | 個人PJのため99%以上（Vercelマネージドサービス活用） |
| セキュリティ | 年収等の入力情報はセッション内のみ保持・DB保存なし（Phase1） |
